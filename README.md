# Novella Bind Score

Full challenge description: [novella_challenge.md](novella_challenge.md)

---

## Problem Overview

Build a `bind_score(submission_id, t)` that ranks insurance submissions by their likelihood of converting (binding). The score is used for **broker effort prioritization** — higher-scored submissions get worked first.

The score is evaluated at `t ∈ {0, 7, 30}` days since submission creation. Earlier predictions are more valuable operationally, so the model must be useful at all three snapshots.

---

## Environment Setup

```bash
conda env create -f environment.yml
conda activate novella
```

To run the notebooks, also register the kernel:

```bash
python -m ipykernel install --user --name novella --display-name "novella"
```

---

## Running Inference

Score a single submission at all time windows:

```bash
python src/predict.py --submission-id 67
```

Score a single submission at a specific time window:

```bash
python src/predict.py --submission-id 67 --t 7
```

Score all submissions:

```bash
python src/predict.py
```

Output is sorted by `t` then `score` descending. The saved model at `output/model/bind_score.json` is used by default.

Custom paths:

```bash
python src/predict.py \
    --model-path output/model/bind_score.json \
    --submissions data/features_submissions.csv \
    --events data/features_events.csv
```

### Training

To retrain the model from scratch (runs LOO-CV, saves evaluation plots, and serialises the final model):

```bash
python src/train.py
```

Custom paths:

```bash
python src/train.py \
    --submissions data/features_submissions.csv \
    --events data/features_events.csv \
    --output_dir output/model_eval_metrics \
    --model_dir output/model
```

Training prints LOO-CV metrics to stdout and saves three figures to `output_dir`. The final model is written to `model_dir/bind_score.json`.
